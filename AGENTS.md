# AGENTS: terraform

## OVERVIEW

Homelab infrastructure-as-code monorepo. Provisions Proxmox LXC/VM fleet, networking, monitoring, and external services via Terraform workspaces with 1Password secret injection and GitHub Actions CI/CD.

- **Domain**: jclee.me | **Subnet**: 192.168.50.0/24 | **PVE host**: .100
- **Workspaces**: 19 directories (10 active TF, 9 template-only)
- **Providers**: bpg/proxmox ~>0.94, 1Password/onepassword ~>3.2, grafana/grafana ~>4.0, elastic/elasticstack ~>0.13, cloudflare/cloudflare ~>5.0, integrations/github ~>6.6, synology-community/synology ~>0.6

## CONVENTIONS

### Config Pipeline

`hosts.tf` SSoT ŌåÆ `module.hosts` ŌåÆ `config-renderer` ŌåÆ `.tftpl` templates ŌåÆ `configs/` rendered outputs ŌåÆ cloud-init deploy. **Never hand-edit `configs/`** ŌĆö regenerate via `terraform apply`.

### State Management

All workspaces use `backend "local" {}`. State files committed to git. CI concurrency groups prevent parallel applies per workspace.

### CI/CD

- Self-hosted runner on LXC 101. `make apply` disabled locally ŌĆö all applies via CI.
- PR triggers `terraform plan`. Merge to master triggers `terraform apply`.
- Drift detection: Mon-Fri 00:00 UTC via scheduled workflow.
- GitHub Actions SHA-pinned with `# vN` comment. Never use mutable tags.

### Secrets

1Password ŌåÆ `onepassword-secrets` module ŌåÆ `.tftpl` templates ŌåÆ `.env` files on hosts. Connect Server at 192.168.50.112:8090. Never commit `.tfvars`, `.env`, `.tfstate`, or API keys.

### Build System

Bazel (Google3 style). Every directory has `BUILD.bazel` + `OWNERS`. Ownership via hierarchical OWNERS files with CODEOWNERS for GitHub enforcement.

### Module Architecture

- `modules/proxmox/` ŌĆö LXC/VM/cloud-init provisioning primitives, `config-renderer`
- `modules/shared/` ŌĆö cross-workspace: `onepassword-secrets`
- Template-only workspaces: `docker-compose.yml.tftpl` + `.env.tftpl` rendered by config-renderer

## ANTI-PATTERNS

- **NO** hand-editing `configs/` or `tf-configs/` ŌĆö always regenerate via TF
- **NO** local `make apply` ŌĆö deploy via CI/CD only
- **NO** committing `.tfvars`, `.env`, `.tfstate`, or secrets
- **NO** hardcoded IPs ŌĆö use `hosts.tf` variables
- **NO** mutable action tags (`@v4`) ŌĆö SHA-pin all GitHub Actions
- **NO** Docker socket mounting in any workspace (CVE-2025-9074)
- **NO** manual LXC/VM resource changes in Proxmox UI ŌĆö Terraform-managed

## COMMANDS

```bash
make test                  # Run all tests
make lint                  # Lint all workspaces
make verify                # Full verification (lint + test + plan)
make fmt                   # Format all .tf files

```

## REVIEW GUIDELINES

- Verify no hardcoded IPs ŌĆö use variables from `hosts.tf`.
- Verify secrets use 1Password `op://` references, never plaintext.
- Verify CI workflows SHA-pin all actions with `# vN` comment.
- Verify config changes go through templates, not rendered `configs/`.
- PR size ~200 LOC max. Squash merge only.
- Conventional commits: `type(scope): summary`.
