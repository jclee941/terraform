# AGENTS: terraform

## OVERVIEW

Homelab infrastructure-as-code monorepo. Provisions Proxmox LXC/VM fleet, networking, monitoring, and external services via Terraform workspaces with 1Password secret injection and GitHub Actions CI/CD.

- **Domain**: jclee.me | **Subnet**: 192.168.50.0/24 | **PVE host**: .100
- **Workspaces**: 15 directories (7 active TF, 8 template-only)
- **Providers**: bpg/proxmox ~>0.94, 1Password/onepassword ~>3.2, grafana/grafana ~>3.0, elastic/elasticstack ~>0.10, cloudflare/cloudflare ~>4.0

## CONVENTIONS

### Config Pipeline

`hosts.tf` SSoT → `module.hosts` → `config-renderer` → `.tftpl` templates → `configs/` rendered outputs → cloud-init deploy. **Never hand-edit `configs/`** — regenerate via `terraform apply`.

### State Management

All workspaces use `backend "local" {}`. State files committed to git. CI concurrency groups prevent parallel applies per workspace.

### CI/CD

- Self-hosted runner on LXC 101. `make apply` disabled locally — all applies via CI.
- PR triggers `terraform plan`. Merge to master triggers `terraform apply`.
- Drift detection: Mon-Fri 00:00 UTC via scheduled workflow.
- GitHub Actions SHA-pinned with `# vN` comment. Never use mutable tags.

### Secrets

1Password → `onepassword-secrets` module → `.tftpl` templates → `.env` files on hosts. Connect Server at 192.168.50.112:8090. Never commit `.tfvars`, `.env`, `.tfstate`, or API keys.

### Build System

Bazel (Google3 style). Every directory has `BUILD.bazel` + `OWNERS`. Ownership via hierarchical OWNERS files with CODEOWNERS for GitHub enforcement.

### Module Architecture

- `modules/proxmox/` — LXC/VM/cloud-init provisioning primitives
- `modules/shared/` — cross-workspace: `config-renderer`, `onepassword-secrets`, `host-variables`
- Template-only workspaces: `docker-compose.yml.tftpl` + `.env.tftpl` rendered by config-renderer

## ANTI-PATTERNS

- **NO** hand-editing `configs/` or `tf-configs/` — always regenerate via TF
- **NO** local `make apply` — deploy via CI/CD only
- **NO** committing `.tfvars`, `.env`, `.tfstate`, or secrets
- **NO** hardcoded IPs — use `hosts.tf` variables
- **NO** mutable action tags (`@v4`) — SHA-pin all GitHub Actions
- **NO** Docker socket mounting in any workspace (CVE-2025-9074)
- **NO** manual LXC/VM resource changes in Proxmox UI — Terraform-managed

## COMMANDS

```bash
make plan SVC=pve          # Plan specific workspace
make test                  # Run all tests
make lint                  # Lint all workspaces
make verify                # Full verification (lint + test + plan)
make fmt                   # Format all .tf files
make validate SVC=pve      # Validate specific workspace
```

## REVIEW GUIDELINES

- Verify no hardcoded IPs — use variables from `hosts.tf`.
- Verify secrets use 1Password `op://` references, never plaintext.
- Verify CI workflows SHA-pin all actions with `# vN` comment.
- Verify config changes go through templates, not rendered `configs/`.
- PR size ~200 LOC max. Squash merge only.
- Conventional commits: `type(scope): summary`.
